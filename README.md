## Описание проекта

**"MEMO BATTLE"** — это классическая игра на развитие памяти, адаптированная для 2-4 игроков в режиме реального времени по локальной сети. 

Игроки по очереди открывают карточки. Если изображения совпадают — игрок получает очко. Если нет — ход переходит сопернику. Побеждает тот, кто соберет больше пар.

## Функциональность
1. **Архитектура:** Клиент-серверная архитектура с MVC-подходом. Сервер управляет игровой логикой, клиенты — отображением и взаимодействием.
2. **Сеть:** TCP сокеты с Java NIO (Non-blocking I/O). Поддержка до 4 игроков одновременно. Асинхронная обработка через селекторы.
3. **Многопоточность:** Со стороны сервера - неблокирующий ввод-вывод в главном потоке, со стороны клиента -  отдельный поток для чтения данных, UI-поток JavaFX
4. **Графика:** JavaFX с кастомными компонентами:
   * Анимированные карточки с переворотом
   * Аватары игроков
   * CSS-стилизация
   * 18 пар предзагруженных изображений мемов
5. **Протокол:** Текстовый протокол: ТИП|ПАРАМЕТР1|ПАРАМЕТР2, 10 типов сообщений

## Технологический стек

*   **Язык:** Java (JDK 8+)
*   **Сборка:** Maven
*   **GUI:** JavaFX

## Протокол обмена данными

Протокол представляет собой текстовый формат для обмена данными между клиентом и сервером игры. Используется упрощенная структура с разделителями для повышения производительности и минимального размера пакетов.

| Код | Тип сообщения | Описание |
|---|---|---|
| 0 | `TYPE_SYSTEM` | Системные сообщения |
| 1 | `TYPE_CONNECT` | Подключение игрока |
| 2 | `TYPE_CARD_OPEN` | Открытие карточки |
| 3 | `TYPE_CHAT_MESSAGE` | Сообщение в чат |
| 4 | `TYPE_GAME_OVER` | Завершение игры |
| 5 | `TYPE_ERROR` | Ошибки |
| 6 | `TYPE_GAME_STATE` | Состояние игры (JSON) |
| 7 | `TYPE_START_GAME` | Начало игры |
| 8 | `TYPE_PLAYER_TURN` | Смена хода |
| 9 | `TYPE_GAME_RESET` | Сброс игры |

## Запуск и установка

### 1. Клонирование и сборка

```
git clone https://github.com/aisinast/sem-work-2-ORIS.git
cd sem-work-2-ORIS
mvn clean package
```

### 2. Запуск
Сервеная часть запускается автоматически при успешном создании игровой комнаты
```
mvn javafx:run
```
